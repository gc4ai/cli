# gc4ai — OSS gc4.ai CLI

> **Generated from the [`gc4ai/openapi`](https://github.com/gc4ai/openapi) contract**
> (via the gc4.ai SDK toolchain) — do not hand-edit. Licensed Apache-2.0.

A thin Node CLI over the generated gc4.ai TypeScript client. **Non-streaming** (typed
SSE is a Speakeasy-tier property; the CLI migrates onto the premium `@gc4ai/sdk`
later).

Regenerated by `apps/web/openapi/generate/generate.sh` (target `cli`). It imports
the generated client from the sibling `typescript-sdk/generated` dir at runtime.

```bash
export GC4AI_API_KEY=<your key>
gc4ai models
gc4ai chat "hello" --model openai/gpt-4o-mini
gc4ai complete "Once upon a time" --model meta/llama-3.3-70b
```

A real chat call needs a live API key + a running gc4.ai (or `wrangler dev`)
worker — that path is **gated** (see `apps/web/openapi/PUBLISHING.md`). The
account-free smoke is `pnpm smoke` (syntax-check + `gc4ai help`).
